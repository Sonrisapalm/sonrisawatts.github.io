Guide to Statistical Interpretation

OR Dot Position

The position of the circle on the line shows the **Odds Ratio**. A dot further to the right indicates a much stronger predictive effect on the outcome (e.g., $OR=5.0$).

OR = 1.0 Line

The vertical dashed line (at the far left) is the baseline where the predictor has **no effect** ($OR=1.0$). All findings show a positive association ($OR > 1$).

P-value Indicator

The badge updates its color and text based on the simulated statistical **significance** ($p$). Blue is $p<0.05$, Yellow is $p<0.15$, and Red is $p \ge 0.15$.

Logistic Regression: Predictors of AI Attitudes

Key Findings from Survey Data ($N=44$ and $N=23$)

Analysis shows how emotional comfort, perceived transparency, and prior experience predict trust and demand for explanation in AI systems.

Summary Interpretation

The first set of models (N=44) suggests a **cognitive-affective consistency** in algorithmic trust. Individuals who feel **comfortable with AI in objective roles** are nearly three times more likely to demand an explanation when automation fails, highlighting that rational clarity is highly valued by comfortable users ($OR=2.96, p=.102$). Critically, **Perceived Transparency** significantly increases **Trust in AI for Financial Advice** ($OR=3.80, p=.038$), a powerful, statistically significant finding.

The inferential analysis (N=23) shows that **Prior AI Use** fosters calibrated trust, with experienced users being approximately **twice as likely to trust** AI advice ($OR=2.06, p=.043$). While not statistically significant, the directional trend for the **Age Group 55+** suggests caution, as this group was about $33\%$ less likely to trust AI advice ($OR=0.67, p=.225$).

// --- Utility Functions and Constants --- const MIN_OR = 1.0; const MAX_OR = 5.0; // Defines the visual scale maximum /** * Calculates the position of the Odds Ratio dot on a visual scale (1.0 to 5.0). * @param {number} or - The Odds Ratio value. * @returns {number} The position as a percentage (0 to 100). */ function oddsRatioToPosition(or) { if (or <= MIN_OR) return 0; if (or >= MAX_OR) return 100; // Map the value from [MIN_OR, MAX_OR] to [0, 100] return ((or - MIN_OR) / (MAX_OR - MIN_OR)) * 100; } /** * Simulates a P-value based on the Odds Ratio (stronger OR = lower P-value). * @param {number} or - The Odds Ratio value. * @returns {number} The simulated P-value. */ function simulatePValue(or) { // Normalized OR from 0 to 1 based on the MIN_OR to MAX_OR range const normalizedOR = Math.min(1, Math.max(0, (or - MIN_OR) / (MAX_OR - MIN_OR))); // Simulates an inverse relationship: as OR increases, P decreases significantly const maxP = 0.999; const minP = 0.001; // Use an exponential decay to make P-value drop quickly as OR increases const pValue = maxP * Math.exp(-3 * normalizedOR) + minP; return Math.min(maxP, Math.max(minP, pValue)); } /** * Provides a plain text and color interpretation for the P-value. * @param {number} p - The P-value. * @returns {Object} { text, color, bg } */ function getPValueInterpretation(p) { if (p < 0.05) { return { text: 'Statistically Significant', color: 'text-primary', bg: 'bg-primary-light' }; } else if (p < 0.15) { return { text: 'Approaching Significance', color: 'text-yellow-700', bg: 'bg-yellow-100' }; } else { return { text: 'Not Significant', color: 'text-red-700', bg: 'bg-red-100' }; } } /** * Generates the HTML for a single finding card. * @param {Object} finding - The finding data object. * @returns {string} The HTML string for the card. */ function createFindingCard(finding) { const orPosition = oddsRatioToPosition(finding.odds_ratio); const percentageIncrease = Math.round((finding.odds_ratio - 1) * 100); const pValueStatus = getPValueInterpretation(finding.p_value); const color = colorsMap[finding.theme]; // Conditional rendering for the interactive slider const sliderHtml = finding.interactive ? `
$1.0$ (Baseline) $5.0$ (Strong)
` : ''; // This is the complete, enhanced card template return `

`; } /** * Updates the visual and textual elements of a card based on a new Odds Ratio. * This function is now generalized to accept the card ID. * @param {string} findingId - The ID of the finding card (e.g., 'finding1'). * @param {string} orString - The new Odds Ratio value (as a string). */ function updateCard(findingId, orString) { const newOR = parseFloat(orString); const findingIndex = findingsData.findIndex(f => f.id === findingId); if (findingIndex === -1) return; // 1. Calculate new derived values const newPosition = oddsRatioToPosition(newOR); const newPValue = simulatePValue(newOR); const pValueStatus = getPValueInterpretation(newPValue); const percentageIncrease = Math.round((newOR - 1) * 100); const color = colorsMap[findingsData[findingIndex].theme]; // 2. Update visual dot position document.getElementById(`${findingId}-dot`).style.left = `${newPosition}%`; document.getElementById(`${findingId}-dot`).title = `Odds Ratio: ${newOR.toFixed(2)}`; // 3. Update main OR and percentage display document.getElementById(`${findingId}-or-value`).textContent = newOR.toFixed(2); document.getElementById(`${findingId}-percent`).textContent = `${percentageIncrease}%`; // 4. Update P-value badge (text and color) const badge = document.getElementById(`${findingId}-p-badge`); badge.textContent = pValueStatus.text; badge.title = `P-value: ${newPValue.toFixed(3)}`; // Remove all existing color classes and add the new ones for the badge badge.className = 'p-value-badge shadow-md'; // Base classes badge.classList.add(pValueStatus.bg, pValueStatus.color); // 5. Update hidden details table document.getElementById(`${findingId}-details-or`).textContent = newOR.toFixed(3); const detailsP = document.getElementById(`${findingId}-details-p`); detailsP.textContent = newPValue.toFixed(3); // Update details P-value color detailsP.className = 'px-3 py-2 text-right font-extrabold'; // clear existing color classes detailsP.classList.add(pValueStatus.color); } /** * Renders all finding cards. */ function renderDashboard() { const container = document.getElementById('findings-container'); let cardsHtml = ''; findingsData.forEach(finding => { cardsHtml += createFindingCard(finding); }); container.innerHTML = cardsHtml; } /** * Toggles the visibility of the detailed analysis section for a card with a smooth transition. * @param {string} id - The ID of the finding card. */ function toggleDetails(id) { const detailsWrapper = document.getElementById(`${id}-details-wrapper`); const button = document.querySelector(`#${id} .details-button`); const arrow = document.getElementById(`${id}-arrow`); const isClosed = button.getAttribute('data-state') === 'closed'; if (isClosed) { detailsWrapper.classList.add('open'); button.textContent = 'Hide Full Statistical Details '; button.setAttribute('data-state', 'open'); arrow.classList.add('rotate-180'); } else { detailsWrapper.classList.remove('open'); button.textContent = 'Show Full Statistical Details '; button.setAttribute('data-state', 'closed'); arrow.classList.remove('rotate-180'); } // Re-append the arrow SVG after text content change button.appendChild(arrow); } // Expose functions globally for the onclick/oninput attributes window.toggleDetails = toggleDetails; window.updateCard = updateCard; // Generalized update function // Initial render window.onload = renderDashboard;
